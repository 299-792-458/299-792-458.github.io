# AI와 합주하다

새해 첫날에는 팀과 함께 "바이브 코딩 세션"을 열었다. 화려한 은유 대신, "AI와 사람이 어떻게 생산성을 나눠 갖는가"를 코드 단위로 실험한 시간이었다. 프롬프트 세트를 파트별로 모듈화해 IDE 액션과 묶어 두니, 커서 이동과 코드 생성이 리듬처럼 반복되며 작업 흐름이 안정됐다.

<pre style="color:#5AA7FF;font-family:var(--font-mono,monospace);background:none;border:none;padding:0;margin:1em 0">
"커서는 악기가 되고, 프롬프트는 리듬이 된다."
</pre>

![라이브 코딩 세션의 감각을 기록한 사진](/log/images/ai-era.webp)

세션의 핵심은 피드백 루프였다. 모델이 제안한 코드에는 바로 정량 지표를 붙이고, 설계가 흐려지면 3줄 요약을 강제해 요구사항을 다시 정렬했다. 커밋 훅에는 실험적인 린터를 추가해, 즉석에서 나온 아이디어라도 소나큐브 규칙과 보안 체크를 최소 수준으로 통과하도록 만들었다.

정리된 규칙은 플레이북에 바로 반영했다. 반복 작업이 느려지면 AI에게 테스트 작성과 로그 포맷팅을 위임했고, 브랜치 병합 전에는 15분짜리 집중 세션을 배치해 TODO와 리스크를 정리했다. 오후에는 이 흐름으로 프로토타입을 완성해 팀 채널에 공유했고, 동료들은 각자 검증 시나리오와 개선 방향을 이어 붙였다.

올해 목표는 이 시스템을 일상 업무로 완전히 편입하는 것이다. 매일 아침 20분짜리 세션으로 프롬프트 라이브러리를 리팩터링하고, 테스트-배포 파이프라인의 템포를 맞춘다. 다음 분기 안에는 이 플레이북을 팀 공용 문서로 정리해, 새로 합류한 동료도 같은 루틴으로 AI를 사용할 수 있게 만들 예정이다.
